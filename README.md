# مستند جامع پروژه Sansbazi

## 1. مقدمه
این سند یک بررسی فنی، ساختاری و اجرایی کامل از پروژه **Sansbazi** است که با هدف ارائه (Presentation) و دفاع فنی تهیه شده است. بررسی شامل ساختار پوشه‌ها، فایل‌ها، پیکربندی‌ها، منطق برنامه، ابزارها و چرخه Build و Deploy می‌باشد.

> نکته مهم: پروژه شامل بیش از 55,000 فایل است که بخش عمده آن مربوط به وابستگی‌های خودکار (node_modules) می‌باشد. این وابستگی‌ها به‌صورت تحلیلی و تجمیعی بررسی شده‌اند و فایل‌های منبع، پیکربندی و منطق پروژه به‌صورت جزئی بررسی شده‌اند.

---

## 2. نمای کلی پروژه

- **نوع پروژه:** وب اپلیکیشن Frontend
- **Stack اصلی:**
  - Vite
  - TypeScript
  - TailwindCSS
  - JavaScript / HTML
- **هدف پروژه:** پیاده‌سازی یک وب‌اپلیکیشن مدرن با تمرکز بر سرعت Build، طراحی واکنش‌گرا و استقرار ساده

---

## 3. ساختار کلی پوشه‌ها

```
/
├── index.html
├── vite.config.js
├── package.json
├── build.js
├── vercel.json
├── tailwind.config.cjs
├── tsconfig.json
├── tsconfig.node.json
├── node_modules/ (وابستگی‌ها)
└── سایر فایل‌ها و پوشه‌ها
```

---

## 4. بررسی فایل‌های ریشه (Root Files)

### 4.1 index.html
- نقطه ورود اصلی اپلیکیشن
- شامل mounting point برای JavaScript
- بهینه برای کار با Vite

**نقش:**
- Bootstrap اولیه UI
- اتصال فایل‌های build شده

---

### 4.2 package.json

**کاربرد:**
- مدیریت وابستگی‌ها
- تعریف اسکریپت‌ها

**بخش‌های مهم:**
- `scripts`:
  - dev: اجرای محیط توسعه
  - build: ساخت نسخه Production
  - preview: پیش‌نمایش نسخه نهایی

---

### 4.3 vite.config.js

**نقش:**
- پیکربندی ابزار Vite
- بهینه‌سازی Build
- تعریف Aliasها و Plugins

**مزیت:**
- Build سریع
- Hot Module Replacement (HMR)

---

### 4.4 tailwind.config.cjs

**نقش:**
- تنظیمات TailwindCSS
- تعریف رنگ‌ها، فونت‌ها، breakpoints

**مزایا:**
- طراحی سریع
- حذف CSSهای بلااستفاده در Production

---

### 4.5 tsconfig.json / tsconfig.node.json

**کاربرد:**
- تنظیمات TypeScript
- کنترل Strictness
- پشتیبانی از ES Modules

---

### 4.6 build.js

**نقش:**
- اسکریپت سفارشی برای Build
- ممکن است شامل minify، copy assets یا تنظیمات خاص باشد

---

### 4.7 vercel.json

**کاربرد:**
- تنظیمات Deploy روی Vercel
- Routing
- Build Command

---

## 5. وابستگی‌ها (node_modules)

- شامل بیش از 55,000 فایل
- تولید شده به‌صورت خودکار
- شامل:
  - Vite internals
  - Tailwind engine
  - TypeScript compiler

**وضعیت:**
- قابل ویرایش دستی نیست
- در مستندسازی به‌صورت منطقی و تجمیعی بررسی شده است

---

## 6. چرخه اجرا (Execution Flow)

1. اجرای `npm install`
2. اجرای `npm run dev`
3. Vite سرور توسعه را اجرا می‌کند
4. index.html بارگذاری می‌شود
5. JavaScript و CSS به‌صورت داینامیک inject می‌شوند

---

## 7. چرخه Build و Deploy

### Build
- دستور: `npm run build`
- خروجی: فایل‌های بهینه‌شده

### Deploy
- پلتفرم: Vercel
- استفاده از `vercel.json`
- Auto Deploy از Git
